XSS: Overprivileged ContextBridge

Security Feature Checklist:
  • NodeIntegration: false ✅
  • ContextIsolation: true ✅
  • Sandbox: false ❌

Guide: What is happening?

This page demonstrates an XSS vulnerability in the renderer process with privileged Node.js or Electron APIs exposed. The renderer has access to privileged APIs via a custom preload script (such as window.systemAPI), even though nodeIntegration is false and contextIsolation is true.

Example payload: 
  • <img src=x onerror="window.systemAPI.runCommand('open -a Calculator')">
    (runs Calculator on macOS)
Impact: An attacker can execute arbitrary JavaScript and also access the file system, OS, or main process APIs, leading to full system compromise. This goes beyond browser-like attacks and allows actions like reading/writing files, executing system commands, or exfiltrating sensitive data from the user's machine.

Note: This demonstrates why least privilege is critical when exposing APIs to the renderer, even with Electron's security features enabled.
