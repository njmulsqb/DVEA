XSS: System APIs Exposed

This page demonstrates an XSS vulnerability in the renderer process with privileged Node.js or Electron APIs exposed.

Explanation: This is a reflected XSS vulnerability, but the renderer has access to privileged Node.js/Electron APIs via a custom preload script (such as window.systemAPI), even though nodeIntegration is false and contextIsolation is true. The vulnerability is caused by exposing over-privileged APIs to the renderer context.

Impact: An attacker can execute arbitrary JavaScript and also access the file system, OS, or main process APIs, leading to full system compromise. This goes beyond browser-like attacks and allows actions like reading/writing files, executing system commands, or exfiltrating sensitive data from the user's machine.

Example Payload: 
<img src=x onerror="window.systemAPI.runCommand('open -a Calculator')">
  • Paste the above payload in the input field and submit to demonstrate the vulnerability.
  • Do not auto-execute the payload for safety.

Note: This demonstrates why least privilege is critical when exposing APIs to the renderer, even with Electron's security features enabled.
