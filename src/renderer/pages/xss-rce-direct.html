XSS → RCE (Direct)

Renderer input eval'd in the main process — full Node.js access without disabling sandbox

✓ NodeIntegration: false ✓ ContextIsolation: true ✓ Sandbox: true ✗ IPC: passes input to eval()

Guide: What is happening?

User input is sent from the renderer to the main process via IPC. The main process passes it directly to eval() — no sandbox is disabled and no new window is created. The code executes in the privileged main process with full Node.js access.

Example payloads: 
require('os').platform()
require('child_process').execSync('whoami').toString()
process.env
Impact: Full RCE in the main process — read/write files, execute system commands, exfiltrate environment variables. The checklist shows all renderer security features as enabled: they cannot protect against a vulnerable IPC handler that evaluates untrusted input.
