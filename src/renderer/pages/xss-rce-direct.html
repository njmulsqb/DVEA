XSS to RCE (Direct)

Security Feature Checklist:
  • NodeIntegration: false ✅
  • ContextIsolation: true ✅
  • Sandbox: true ✅

Guide: What is happening?

This demo shows how a renderer XSS vulnerability can be escalated to Remote Code Execution (RCE) in Electron apps. Here, user input from the renderer is sent to the main process via IPC, and the main process executes the input as code using eval.
No sandbox is disabled and no new window is created—the code is executed directly in the main window context.

Example payloads: 
  • require('os').platform()
    (returns OS platform)
  • require('child_process').execSync('whoami').toString()
    (runs OS command)
  • process.env
    (shows environment variables)
Impact: An attacker can execute arbitrary JavaScript and escalate to full Remote Code Execution in the main process, allowing actions like reading/writing files, executing system commands, or exfiltrating sensitive data from the user's machine.

Note: This demonstrates why never pass unsanitized user input to privileged code execution APIs, even with Electron's security features enabled.
