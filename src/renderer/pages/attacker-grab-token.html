Step 2 — Call the unvalidated bridge

The main process get-token handler performs no sender validation. Any page loaded in this window — regardless of origin — can invoke it: 

await window.analyticsAPI.getToken()
Waiting…
Why this works: Electron preload scripts execute for every navigation within a webContents. The main process get-token handler never checks event.senderFrame.url or validates the sender's origin against an allowlist.

Fix: Validate the sender in the IPC handler: if (!event.senderFrame.url.startsWith('file:///.../analytics.html')) return; Or apply a capability-based model — expose only what each page strictly needs, and never expose privileged bridges to windows that may navigate to untrusted content.
