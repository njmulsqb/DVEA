Open Redirect

Unvalidated deep link parameter loads arbitrary URLs in the app window

Guide: What is happening?

This vulnerability is triggered via a custom deep link. The app registers a dvea:// protocol handler with the OS. When launched with a deep link, the main process reads the redirect query parameter and loads it directly — with no validation.

Deep link format:

dvea://?redirect=https://example.com

You can also use the button below to simulate the redirect without crafting a deep link.

Impact: An attacker who convinces a user to click a crafted dvea:// link (e.g., in an email or on a web page) can load any URL inside the app window. Because there is no address bar, the user has no signal that the app is no longer showing trusted content — making this a viable phishing vector.
