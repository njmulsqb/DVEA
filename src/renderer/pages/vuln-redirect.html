Open Redirect

Vulnerability: This demo shows how an Electron app can be vulnerable to open redirects. If user input or parameters are used to construct a redirect without validation, attackers can send users to malicious sites.

How to invoke: In this demo, the vulnerability is triggered via a deep link. Use a deep link in the format dvea://?redirect=https://example.com to redirect the app to the specified URL.

Example: dvea://?redirect=https://evil.com

You can also use the Simulate Redirect button below to test the vulnerability manually.
