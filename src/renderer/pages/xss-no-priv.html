XSS: No Privileged APIs Exposed

Security Feature Checklist:
  • NodeIntegration: false ✅
  • ContextIsolation: true ✅
  • Sandbox: true ✅

Guide: What is happening?

This page demonstrates a classic XSS vulnerability in the renderer process, with no privileged Node.js or IPC APIs exposed. User input is injected into the DOM using innerHTML without sanitization.

Example payload: 
  • <img src=x onerror=alert(1)>
    (shows alert)
Explanation: This is a classic reflected XSS vulnerability. The attacker can execute arbitrary JavaScript in the renderer, but cannot access the file system, OS, or Electron main process APIs. The risk is similar to a web browser XSS.
