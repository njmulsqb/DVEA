XSS: No Privileged APIs Exposed

Renderer-contained XSS — equivalent impact to a browser

✓ NodeIntegration: false ✓ ContextIsolation: true ✓ Sandbox: true

Guide: What is happening?

User input is injected directly into the DOM via innerHTML without sanitization. All Electron security features are enabled, so the attacker is limited to the renderer process — no Node.js or IPC APIs are accessible.

Example payload: 
<img src=x onerror=alert(1)>
Impact: The attacker can execute arbitrary JavaScript in the renderer but cannot access the file system, OS, or Electron main process APIs. The risk is equivalent to a browser XSS — session hijacking, UI redressing, or credential phishing within the renderer context.
