XSS: No Privileged APIs Exposed

This page demonstrates a classic XSS vulnerability in the renderer process, with no privileged Node.js or IPC APIs exposed.

Explanation: This is a classic reflected XSS vulnerability. User input is directly injected into the DOM using innerHTML without sanitization. However, since no privileged Node.js or IPC APIs are exposed to the renderer, the attacker is limited to what the browser context allows (e.g., stealing cookies, defacing the UI, or phishing).
Impact: The attacker can execute arbitrary JavaScript in the renderer, but cannot access the file system, OS, or Electron main process APIs. The risk is similar to a web browser XSS.
