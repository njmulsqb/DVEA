Insecure File Write (IPC Abuse)

Renderer-supplied path and content written without validation

✓ NodeIntegration: false ✓ ContextIsolation: true ✓ Sandbox: true ✗ IPC: no path or content validation

Guide: What is happening?

The form below lets you specify any file path and content. When you click "Save File", the data is sent to the main process, which writes the file without any validation or restriction.

Example — overwrite a sensitive file: 
Path:    /tmp/dvea-demo.txt
Content: Hello from DVEA!
Try paths like ~/.bashrc or package.json to see the full impact.
Impact: Overwrite or create sensitive files anywhere the app has write access — ~/.bashrc, package.json, /tmp/evil.js. Drop malicious scripts, modify app config, or gain persistence if the app runs with elevated privileges.
