Insecure File Write (IPC Abuse)

Security Feature Checklist:
  • NodeIntegration: false ✅
  • ContextIsolation: true ✅
  • Sandbox: true ✅

Guide: What is happening?

This demo shows how insecure IPC can allow arbitrary file writes from the renderer process. The form below lets you specify any file path and content. When you click "Save File", the data is sent to the main process, which writes the file without any validation or restriction.

Example usage: 
  • Path: /tmp/dvea-demo.txt
Content: Hello from DVEA!
    (creates or overwrites a file)
Impact: Overwrite or create sensitive files anywhere the app has write access (e.g., ~/.bashrc, package.json, /tmp/evil.js). Drop malicious scripts, modify app config files, or gain persistence if the app runs with higher privileges.
