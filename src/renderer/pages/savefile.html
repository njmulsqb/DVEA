Insecure File Write (IPC Abuse)

Guide: What is happening?

This demo shows how insecure IPC can allow arbitrary file writes from the renderer process. The form below lets you specify any file path and content. When you click "Save File", the data is sent to the main process, which writes the file without any validation or restriction.

How can this be exploited?

  • Overwrite or create sensitive files anywhere the app has write access (e.g., ~/.bashrc, package.json, /tmp/evil.js).
  • Drop malicious scripts or modify app config files.
  • Persistence or privilege escalation if the app runs with higher privileges.
Try it: The fields below are prefilled for a quick test. Change the path/content as you wish.
