openExternal Abuse (Protocol Handling)

Guide: What is happening?

This demo shows how unvalidated use of shell.openExternal can allow the renderer to open any URL or protocol handler on the system. The form below lets you specify any URL. When you click "Open Link", the URL is sent to the main process, which opens it using the system handler without any validation or restriction.

How can this be exploited?

  • Trigger the user's browser to open a phishing or malware site.
  • Abuse custom protocol handlers (e.g., mailto:, tel:, slack://, vscode://).
  • Exploit OS-level protocol handler vulnerabilities.
