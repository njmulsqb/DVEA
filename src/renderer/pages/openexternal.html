openExternal Abuse

Unvalidated URLs passed to shell.openExternal() trigger arbitrary OS protocol handlers

Guide: What is happening?

The form below lets you specify any URL. When you click "Open Link", it is sent to the main process which calls shell.openExternal(url) without any validation or allowlist.

How can this be exploited?
  • Trigger the user's browser to open a phishing or malware site.
  • Abuse custom protocol handlers — mailto:, tel:, slack://, vscode://.
  • Exploit OS-level protocol handler vulnerabilities.
Example URLs to try: 
https://example.com
mailto:attacker@example.com
file:///etc/passwd
Impact: An attacker can open arbitrary URLs or system protocol handlers, potentially triggering phishing, malware downloads, or OS-level exploits. Custom protocol handlers can be abused to open local files or pivot to further attacks.

Fix: Validate against a strict protocol allowlist (https: only, or a named set of known-safe schemes) — never use a denylist, as it is easy to bypass with lesser-known schemes.
