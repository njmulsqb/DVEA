# Contributing to DVEA

Thank you for your interest in contributing to the Damn Vulnerable ElectronJS App (DVEA)!

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Clone your fork** and set up the project using:
   ```sh
   npm install
   ```
3. **Make your changes** in a new branch. Please keep your changes focused and well-documented.
4. **Test your changes** to ensure they work as expected and do not break existing functionality.
5. **Commit your changes** with clear, descriptive commit messages.
6. **Push to your fork** and submit a pull request (PR) to the `main` branch of this repository.

## Code Style
- Use consistent formatting and follow the existing code style.
- Use descriptive variable and function names.
- Add comments where necessary for clarity.

## Pull Request Guidelines
- Clearly describe the purpose of your PR.
- Reference any related issues in your PR description.
- Ensure your PR passes all tests and builds successfully.
- Be responsive to feedback and requested changes.

## Reporting Issues
If you find a bug or have a feature request, please open an issue with:
- A clear and descriptive title
- Steps to reproduce (for bugs)
- Expected and actual behavior
- Any relevant screenshots or logs

## Code of Conduct
Please be respectful and considerate in all interactions. Harassment or abusive behavior will not be tolerated.

Thank you for helping improve DVEA!
